CONTEXT7-VERIFIED FACTS (Streamlit Community Cloud docs, fetched 2026-06-16):
- Dependency-file priority order is EXACTLY: uv.lock -> Pipfile -> environment.yml -> requirements.txt -> pyproject.toml (confirms CONTEXT.md D-10). The committed uv.lock is detected first and installed with uv.
- CRITICAL precedence rule: Cloud "searches for dependency files starting in the directory where your entrypoint file is located, then in the root of your repository" and "files in the entrypoint directory take precedence over files in the repository root." Entrypoint = app/Home.py, so a stray dependency file in app/ (e.g. app/requirements.txt or app/uv.lock) WOULD SHADOW the root uv.lock. This is exactly why D-12's "no stray dependency file inside app/" check matters.
- Only ONE dependency file is used per app — the first encountered by priority + directory order.
- Cloud copies all repo files and runs `streamlit run` from the repo root; entrypoint files may live anywhere (here: app/Home.py).
- secrets.toml is NOT committed; it is pasted into the Cloud app's Secrets manager (Edit Secrets) separately.
