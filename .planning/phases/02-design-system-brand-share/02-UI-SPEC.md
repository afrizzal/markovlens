## Typography

### Active Phase 02 type scale

The executor applies **exactly these 4 sizes and 2 weights** in Phase 02. No other sizes or
weights should appear in Phase 02 HTML/CSS output.

| Role | Size | CSS Variable | Weight | Line Height | Notes |
|------|------|-------------|--------|-------------|-------|
| Heading (page title / h1) | 32px | `--fs-32` | 600 | 1.2 | Page `<h1>` via `.t-h1` |
| Heading (card / section / panel) | 18px | `--fs-18` | 600 | 1.2 | KPI card labels, chart container titles, panel headings via `.t-h3` |
| Body / control labels / small | 14px | `--fs-14` | 400 | 1.5 | `body` default, control labels, table cells, tab labels, description paragraphs |
| Extra small / label / axis | 12px | `--fs-12` | 400 | 1.4 | Uppercase field labels (`.t-label`), badge text, axis tick labels |

Active weights: **regular (400)** — body, control labels, small text. **Semibold (600)** — all headings, emphasis labels.

Letter spacing:
- Headings (`.t-h1`, `.t-h3`): `-0.01em`
- Uppercase labels (`.t-label`): `+0.04em` with `text-transform: uppercase`

Mono usage: all numeric values in KPI cards, table cells with class `.num`, axis tick labels, heatmap cell values, model IDs (m1/m2/m3). Apply `font-variant-numeric: tabular-nums`. Mono is a font-family choice, not an additional size/weight — it uses the same sizes from the active scale above.

| Stop | Value | Meaning |
|------|-------|---------|
| `--chart-seq-1` | `#EEF2FF` | Near-zero probability (light lavender) |
| `--chart-seq-2` | `#C7D2FE` | Low probability |

### Interaction gate (D-11, D-15)

| State | Behavior |
|-------|----------|
| Dataset selected, no forecast yet | Heatmap (Transition Matrix tab) renders immediately. Overview / Monte Carlo / Model Comparison show `empty_state`. |
| "Run Forecast" clicked | All three models computed; spinner shown; results populate all tabs. |
| Model picker changes (post-forecast) | Overview stacked-area and heatmap update to show selected model's primary result. Comparison tab still shows all three. |
| Dataset changes (post-forecast) | Forecast results cleared; heatmap re-renders; tabs revert to empty state until re-run. |

- Legend below chart: brand name + 8×8px color swatch, horizontal, centered

Brand color assignment: `chart-1` through `chart-6` in dataset state order (index-stable).

**Right column: Stationary distribution panel**

`st.container` styled as `.card .accent-card` (accent = `var(--color-primary)`).

Title: `"Long-run equilibrium"` (`.t-h3`)
Subtitle/caveat: `"Assumes today's transition rates hold indefinitely — a what-if, not a prediction."` (`.t-xs .t-ter`, italic or secondary color)

- Always computed from the **m1 (constant) matrix** regardless of the primary model picker (D-13)
- Method: dominant left eigenvector of P; fallback power-iteration (Pⁿ until convergence); fallback `st.warning` if both fail
- Chart: `go.Bar`, horizontal, bars colored by `chart-1..n`, y-axis = state labels, x-axis = probability [0, 1]
- Values annotated on bars as `"{v*100:.1f}%"` in mono
- `use_container_width=True`, height 280px
- If fallback warning shown: `st.warning("Could not compute stationary distribution for this matrix.")`

#### Tab: Transition Matrix

Full-width layout.

- `transition_heatmap` component (see §2 above)
- For m2/m3: show `period_label` subtitle with most-recent period index
- Below heatmap: sparsity summary (if any cells < 20 obs): `st.warning(f"⚠ {n_sparse} cells have fewer than 20 observations — estimates are noisy. Consider merging sparse states or collecting more data.")`
- Heatmap renders on dataset-select without requiring "Run Forecast" click (cheap DB query)

#### Tab: Monte Carlo

Two-column layout: `st.columns([0.65, 0.35], gap="large")`.

**Left: Fan chart controls + chart**

Secondary control strip within tab (separate from page-level strip):
- `st.selectbox("Start brand", options=state_labels)` — selects which brand's probability distribution to simulate
- `st.slider("Time horizon", 1, 24, 12, format="%d mo")` — local override, default from page control
- `st.slider("Simulations", 1000, 50000, 10000, step=1000, format="%dk")` — note: "more = smoother, slower"
- Seed row: `st.number_input("Seed", value=42)` + `st.button("Randomize", icon="🎲")` side by side via `st.columns`
- `st.button("Run Simulation", type="primary")`

Fan chart: `monte_carlo_fan` component (see §3 above).

**Right: Final-state distribution**

`ChartContainer` with title `"Final state distribution"`, subtitle `"Where {brand_label} lands at +{horizon}mo"`.

`go.Bar` (vertical), x = state labels, y = final-state probabilities from Monte Carlo output, color = `chart-1`, annotations showing `"{v*100:.0f}%"` above bars.

#### Tab: Model Comparison

Three model cards at top, then metrics table, then walk-forward backtest table, then interpretation paragraph.

**Model cards (3-column `st.columns`):**

Each card:
- `.card .card-pad` with `box-shadow: 0 0 0 2px var(--color-primary)` on the winning model
- `"Best fit"` badge (success variant) absolutely positioned top-right of winning card
- Model ID in mono `.t-h3`, colored `--color-primary` if recommended else `--color-text-tertiary`
- Model formula in mono `.t-xs .t-sec`
- Mini forecast sparkline: `MiniForecast`-style SVG or `Sparkline` component, height 64px, color = `chart-1` for winner else `--color-text-tertiary`
- Metrics below divider: MAPE in `--fs-32` weight 600 (success color for winner), Brier in 18px weight 600

**Metrics table (`st.dataframe` or custom HTML table):**

Columns: `Model | MAPE ↓ | Brier ↓ | Log-loss ↓`
- Best value per column: **bold** (not per row)
- Cell values: mono tabular-nums, right-aligned
- Header: `.t-xs .t-sec`, 12px, left-aligned

**Walk-forward backtest table:**

Columns: `Period | Actual | m1 | m2 | m3`
- Actual: weight 600
- Model columns: cell background shading = error magnitude, `color-mix(in srgb, var(--color-danger-subtle) {t*100}%, transparent)` where `t = min(1, abs_error / 0.015)`

**Interpretation paragraph:**

`st.markdown(verdict_text)` — auto-generated prose below the backtest table. Template:
> `"**{best_model}** gives the best overall fit (MAPE {best_mape:.2f}%). {reason_sentence} Use the **'How to read these metrics'** expander below for definitions."`

Reason sentence logic (D-12):
- If m1 wins: `"A constant-matrix model works well here — brand-switching rates appear stable over the observed periods."`
- If m2 wins: `"A time-varying model captures shifting brand-switching rates that a constant-matrix m1 cannot."`
- If m3 wins: `"The extended model with growth multiplier better captures changes in total market size alongside shifting switching rates."`

`st.expander("How to read these metrics", expanded=False)`:
```
- **MAPE** (Mean Absolute Percentage Error): average forecast error as a % of the actual value. Lower is better. < 2% is excellent for market share.
- **Brier score**: mean squared error on probability forecasts. Ranges 0–1; lower is better.
- **Log-loss**: penalises confident wrong predictions more heavily. Lower is better.
```

## Page State Machine

| State ID | Trigger | Visual |
|----------|---------|--------|
| `initial` | Page load, dataset not yet selected | Control strip visible; all tabs show `empty_state("No forecast yet")` |
| `dataset_selected` | User picks dataset from selectbox | Heatmap tab renders immediately. Other tabs still show empty state. KPI strip hidden or shows `"—"` values. |
| `running` | "Run Forecast" clicked | `st.spinner("Running forecast across all three models…")` wraps service call |
| `results_loaded` | Forecast completes | KPI strip populated. All four tabs show content. |
| `error_sparse` | `DatasetTooSparseError` raised | `st.error(message)` + `st.info("Try selecting a longer date range or merging states with fewer than 20 observations.")`. Heatmap still renders if matrix was partially built. Forecast tabs remain empty. |
| `error_generic` | Unexpected exception | `st.error(f"Unexpected error: {e}")` + `st.exception(e)` if dev mode + `st.info("If this persists, re-run the seed script or check the logs.")` |
| `mc_running` | "Run Simulation" clicked in Monte Carlo tab | `st.spinner("Running {n_simulations:,} simulations…")` |

## Accessibility

- All interactive controls (buttons, sliders, selects) use Streamlit native — keyboard-navigable by default
- Heatmap cells: Plotly hover text carries all information (no color-only encoding). Sparsity marker uses both color (`#D97706`) and symbol (`⚠`)
- Fan chart: bands differentiated by labeled legend entries (P10/P50/P90), not color alone
- Color contrast: primary `#4338CA` on white `#FFFFFF` = 5.9:1 (WCAG AA ✓). `--color-text-secondary` (`#52525B`) on white = 7.5:1 ✓
- `@media (prefers-reduced-motion: reduce)` disables CSS transitions (from `markov.css`)
- Empty state descriptions are instructional (tell user what to do next)
