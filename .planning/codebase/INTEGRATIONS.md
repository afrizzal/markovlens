## Data Storage

**Databases:**
- **DuckDB (embedded, file-based)**
  - Location: `data/markovlens.duckdb` (via `DUCKDB_PATH` env var)
  - Connection: `core/db/connection.py` (singleton pattern)
  - Client: duckdb 1.1.0+ (Python native)
  - Purpose: Single source of truth for all persistent data
  - Tables (via `core/db/schema.sql`):
    - `datasets` — registered datasets with metadata
    - `transitions` — raw transition observations (entity, period, from_state, to_state)
    - `transition_matrices` — cached computed matrices (matrix_json, observation counts)
    - `simulation_runs` — Monte Carlo result cache (final distributions, quantile paths)
    - `forecasts` — forecast outputs with accuracy metrics
    - `scenarios` — saved what-if scenarios (modified transitions + results)
  - No server required; file-based (gitignored, regenerable from raw data)

**File Storage:**
- **Local filesystem only**
  - Raw data: `data/raw/` (gitignored, sourced from Kaggle)
  - Processed Parquet: `data/processed/` (gitignored, generated by `scripts/seed_data.py`)
  - DuckDB file: `data/markovlens.duckdb` (gitignored)
  - Reports output: (deferred to Phase 04, likely `reports/`)

**Caching:**
- **DuckDB native caching** (via `transition_matrices` and `simulation_runs` tables)
- No Redis, Memcached, or other external cache services
- Cache invalidation via direct DB updates when dataset/model changes
